How to setup Pixel and host it yourself.

Should you get stuck throughout this tutorial, refer to the Official Support Server.

1. Click here to go to the repository. 2. Click the green "Clone or download" button and select Download ZIP.

3. Save it to somewhere easy to access, like Desktop. For a Linux VPS, use something like WinSCP to drag-and-drop the ZIP file.

4. Extract the file (find a tutorial for your Operating System).

5. Install NodeJS and Git for your OS.

6. Go into the scripts folder, choose your OS and run the install-deps file.

7. Rename the credentials_example.json file to credentials.json and edit the file. Change the "token" to your bot's token. You can also change the prefix if you don't like the default one. Paste your ID in the "botOwners" section. To do this, go into Discord, go to Settings go to Appearance and enable the "Developer Mode". Then, right click your user account and click "copy ID".

8. Finally, and here's the tricky part, go to https://mlab.com and register an account on the "free" tier. Create a database. Once it is created, go to the database and click users. Click create new user and name it something memorable like "admin". Add a super secure password and DO NOT click "view only" as that will prevent the bot from being able to edit and create files. Edit the credentials.json file to make it suit your database, for example: "mongodb://admin:s3cur3_p455w0rd@fg187251.mlab.com:94294/super-cool-bot". Save the file.

9. To start the bot, go back into the scripts folder, choose the OS and then execute the start files. TIP: If you're using a command line-based VPS, type cd Pixel-master from @root, then node src/app.js. If you want the bot to stay online without the SSH app being open, type screen -s pixel node src/app.js.

10. The bot should start, if it doesn't, tell us your error in the issues section or join the Discord server: here .
